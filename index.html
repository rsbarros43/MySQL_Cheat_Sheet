Instalação & Configuração

Linux · Repositórios · systemd · my.cnf · Inicialização segura
Básico
Instalação Linux
root@linux
# dnf install mysql-community-server # systemctl enable --now mysqld # systemctl status mysqld # grep 'temporary password' /var/log/mysqld.log # mysql_secure_installation
Parâmetros importantes
ParâmetroUso
innodb_buffer_pool_sizePrincipal cache do InnoDB; normalmente o maior consumidor de memória.
max_connectionsQuantidade máxima de conexões simultâneas.
log_binHabilita binary log para replication e PITR.
server_idIdentificador único em ambientes replicados.
slow_query_logHabilita log de queries lentas.
long_query_timeThreshold para slow query log.
Exemplo my.cnf
[mysqld] server_id=10 datadir=/var/lib/mysql port=3306 default_authentication_plugin=caching_sha2_password innodb_buffer_pool_size=4G innodb_redo_log_capacity=2G log_bin=mysql-bin binlog_expire_logs_seconds=604800 slow_query_log=ON long_query_time=1
Validação
$ mysqladmin -uroot -p ping $ mysql -uroot -p -e "select @@version, @@datadir" $ ss -lntp | grep 3306 $ journalctl -u mysqld -n 100 --no-pager

InnoDB Storage Engine

Buffer Pool · Redo · Undo · MVCC · Locks · Tablespaces
Intermediário
Componentes InnoDB
ItemDescrição
Buffer PoolCache de páginas de dados e índices.
Redo LogGarante durabilidade e crash recovery.
Undo LogPermite rollback e MVCC.
Doublewrite BufferProteção contra partial page writes.
Change BufferOtimiza mudanças em índices secundários.
Adaptive Hash IndexOtimização de lookup para workloads específicos.
Monitorar InnoDB
mysql> show engine innodb status\G mysql> show variables like 'innodb_buffer_pool_size'; mysql> show global status like 'Innodb_buffer_pool%'; mysql> select * from sys.innodb_lock_waits\G
Tablespaces
  • file-per-table — um arquivo .ibd por tabela quando habilitado.
  • system tablespace — arquivo ibdata1 para estruturas internas e, em versões antigas, dados.
  • undo tablespace — armazena undo fora do system tablespace em versões modernas.
  • temporary tablespace — usado para objetos temporários InnoDB.
Locks
LockDescrição
Record lockBloqueio de registro de índice.
Gap lockBloqueio de intervalo entre registros.
Next-key lockRecord + gap lock, comum em REPEATABLE READ.
Metadata lockBloqueio de DDL/estrutura de objetos.
DeadlockCiclo de dependência entre transações; InnoDB escolhe uma vítima.

Backup & Restore

mysqldump · MySQL Shell · MyDumper · XtraBackup · PITR
Intermediário
Estratégias
FerramentaUso
mysqldumpBackup lógico simples, portátil, bom para bancos menores.
mysqlpumpDump lógico com paralelismo em versões compatíveis.
MySQL Shell DumpDump/load moderno e paralelo com util.dumpInstance/util.loadDump.
MyDumperBackup lógico paralelo muito usado em ambientes grandes.
XtraBackupBackup físico online para InnoDB.
Binary LogsBase para point-in-time recovery.
mysqldump
$ mysqldump -uroot -p --single-transaction --routines --triggers --events appdb > appdb.sql $ mysqldump -uroot -p --all-databases --single-transaction --master-data=2 > full.sql $ mysql -uroot -p appdb < appdb.sql
MySQL Shell
$ mysqlsh root@localhost:3306 JS> util.dumpInstance("/backup/full", {threads: 8}) JS> util.dumpSchemas(["appdb"], "/backup/appdb", {threads: 8}) JS> util.loadDump("/backup/appdb", {threads: 8})
PITR com binlog
$ mysqlbinlog --start-datetime="2026-06-12 10:00:00" --stop-datetime="2026-06-12 11:00:00" mysql-bin.000123 > pitr.sql $ mysql -uroot -p < pitr.sql

Performance Tuning

EXPLAIN · Slow Query · Performance Schema · Índices · Observabilidade
Intermediário
Fluxo de análise
EtapaAção
1. SintomaCPU, I/O, lock, query lenta ou saturação de conexões.
2. EvidênciaSlow log, Performance Schema, sys schema e métricas de host.
3. PlanoEXPLAIN, EXPLAIN ANALYZE, cardinalidade e índices.
4. CorreçãoÍndice, rewrite, particionamento, parâmetro ou arquitetura.
5. ValidaçãoComparar tempo, rows examined, locks, I/O e CPU antes/depois.
SQL Tuning
mysql> explain select * from pedidos where cliente_id=10; mysql> explain analyze select * from pedidos where cliente_id=10; mysql> show index from pedidos; mysql> analyze table pedidos;
Slow Query Log
mysql> set global slow_query_log=ON; mysql> set global long_query_time=1; mysql> show variables like 'slow_query_log%'; $ mysqldumpslow -s t -t 20 /var/lib/mysql/*slow*.log
sys schema
mysql> select * from sys.statement_analysis limit 10; mysql> select * from sys.schema_table_statistics order by total_latency desc limit 10; mysql> select * from sys.user_summary_by_statement_latency;

Segurança

Usuários · Roles · TLS · Password Policy · Auditoria
Intermediário
Boas práticas
  • Menor privilégio — conceder apenas o necessário por schema/tabela.
  • Não usar root em aplicação — criar usuários específicos por serviço.
  • TLS — exigir conexão criptografada em rede não confiável.
  • Rotação de senhas — controlar validade e política de senha.
  • Auditoria — usar logs/plugins conforme edição e necessidade regulatória.
Roles e grants
mysql> create role app_readonly; mysql> grant select on appdb.* to app_readonly; mysql> grant app_readonly to 'app'@'%'; mysql> set default role app_readonly to 'app'@'%';
TLS
mysql> show variables like '%ssl%'; mysql> alter user 'app'@'%' require ssl; mysql> status;
Auditoria básica
mysql> show variables like 'general_log'; mysql> set global general_log=ON; mysql> select user, host from mysql.user;
Em produção, general_log pode gerar alto volume. Use com critério.

Componentes
ItemDescrição

  • Garantir usuários administrativos e recovery user.

  • Monitorar quorum, latência e estado dos membros.

Upgrade & Versionamento

5.7 → 8.0 → 8.4 LTS → 9.x · Compatibilidade · Deprecações
Avançado
Trilha recomendada
  • MySQL 5.7 — versão legada, comum em ambientes antigos.
  • MySQL 8.0 — série bugfix antes do modelo LTS/Innovation.
  • MySQL 8.4 LTS — linha LTS moderna para produção estável.
  • MySQL 9.x — linha Innovation/LTS atual conforme disponibilidade.
Para upgrades antigos, valide caminho, compatibilidade, reserved words, authentication plugin e mudanças de data dictionary.
Pré-checks
mysql> select @@version; mysql> show variables like 'sql_mode'; mysql> show variables like 'default_authentication_plugin'; $ mysqlsh -- util checkForServerUpgrade root@localhost:3306
Checklist
  • Backup full validado antes do upgrade.
  • Teste em homologação com cópia realista.
  • Validar aplicação, drivers e conectores.
  • Revisar plugins, charset/collation e autenticação.
  • Planejar rollback e janela de manutenção.
Pós-upgrade
$ systemctl restart mysqld mysql> select @@version; mysql> check table appdb.pedidos; mysql> analyze table appdb.pedidos;

Cloud & MySQL HeatWave

OCI · HeatWave · DB Systems · Backup · Monitoring · Migration
Expert
Cloud DBA
ItemDescrição
MySQL HeatWaveServiço gerenciado MySQL com aceleração analítica HeatWave.
DB SystemInstância gerenciada com compute, storage, backups e rede.
Backup automáticoRetenção e restore gerenciados pela plataforma.
MonitoringMétricas de CPU, memória, conexões, storage e queries.

Migração
  • Inventariar versão, tamanho, engines, usuários e rotinas.
  • Validar compatibilidade 5.7/8.0/8.4.
  • Escolher método: dump lógico, replication, MySQL Shell ou ferramenta cloud.
  • Planejar cutover, rollback e janela.
  • Executar validação de dados, performance e aplicação.
Comandos úteis

Checklist profissional
  • Backups e restore testados.
  • Alertas para storage, conexões, replication e slow queries.
  • Usuários com menor privilégio.
  • Plano de upgrade e patching.
  • Runbook de incidentes e continuidade.
